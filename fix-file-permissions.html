DOCROOT Directory Permissions

The document root directory itself is set to permission 0750.

This means:

  • Owner has full access (read, write, execute)
  • Group can read and execute
  • Others have no access

Why this is important:
This prevents unauthorized users on the server from listing or accessing your website‚Äôs root directory, while still allowing the web server (which runs under the group) to serve the site correctly.

Subdirectories Inside DOCROOT ‚Äì 0755

All normal directories inside the document root use permission 0755.

This means:

  • Owner has full access
  • Group and others can read and execute (but not write)

Why this is required:
Directories must be executable to allow file access through the web server. This ensures the website works normally while preventing unauthorized modification.

Files Inside DOCROOT ‚Äì 0644

All files inside the document root are set to permission 0644.

This means:

  • Owner can read and write
  • Group and others can only read

Why this is safe:
The web server requires read access to files such as HTML, PHP, and images. Write access is restricted to the owner, preventing unauthorized changes.

Multiple Domains & Subdomains

If the account contains a main domain, addon domains, or subdomains, each domain‚Äôs document root is handled separately but follows the same permission rules:

  • DOCROOT directory ‚Üí 0750
  • Subdirectories ‚Üí 0755
  • Files ‚Üí 0644

This ensures security and consistency across all hosted websites.

Mail Directory Permission Rule

The main mail directory must have permission 0751.

This allows:

  • Owner ‚Üí full access (read, write, execute)
  • Group ‚Üí read and execute
  • Others ‚Üí execute only (required for mail services)

Each domain and user folder inside the mail directory also uses 0751, ensuring mail services can function while keeping files protected.

Mail Files ‚Äì 0640

All regular files inside mail folders use permission 0640.

  • Owner can read and write
  • Group can read
  • Others have no access

Special File ‚Äì maildirsize

Path: /mail/domain/user/maildirsize
Permission: 0600

Only the owner can read and write this file. No group or public access is allowed. This file controls mailbox size limits and must remain strictly protected.

ETC Directory Permissions

The /etc directory inside a cPanel user‚Äôs home folder stores critical email configuration files such as routing settings and account controls.

The root /etc directory uses permission 0750, allowing access only to the account owner and required mail services.

Subdirectories inside /etc also use 0750, while files use 0640, ensuring sensitive data is never publicly accessible.

Why these permissions are required:

  • Prevents unauthorized access
  • Ensures stable mail delivery
  • Meets cPanel security standards

Exclude Files & Folders

The Exclude feature allows you to skip specific paths during permission scans and fixes.

Excluded paths are stored in: /home/USERNAME/.permission_exclude.txt

Any excluded item will not be scanned, listed, or modified.

Adding an Exclude

  1. Open Exclude Files / Folders
  2. Enter the relative path
  3. Click Add

Removing an Exclude

  1. Select paths using checkboxes
  2. Click Remove Selected
  3. The path is deleted from the list
